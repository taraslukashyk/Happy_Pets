## 1. Філософія та цілі сайту

Сайт — це не «візитка», а **конверсійний хаб**. На нього людина приходить з Instagram, Google або з офлайн-реклами, і за 60 секунд має:

1. Зрозуміти, що це грумер-салон у Черкасах, який працює з великими породами і робить спа.
2. Побачити ціну і тривалість потрібної послуги.
3. Записатися через Easyweek або написати в Telegram/Instagram.

Усе інше вторинне. Сайт повинен **доганяти те, чого Instagram дати не може**:
- прозорий прайс;
- глибокий SEO-трафік;
- прямий запис без Direct-комунікацій;
- довіра через відгуки та фото до/після;
- повноцінний каталог послуг із фільтрами.

### Ключові завдання

| Що дає сайт | Чого не може Instagram |
|---|---|
| Органічний трафік з Google | Обмежений охопленням IG |
| Повний прайс без скролу stories | Ховає ціни в hylights |
| Пряме бронювання в 2 кліки | Переписка у Direct |
| Фільтри по породах/послугах | Лінійна стрічка |
| Довгий контент (блог, SEO) | Короткі формати |
| Сертифікати, акції, промокоди | Немає комерційних функцій |

## 2. Технологічний стек

Оскільки проект робиться в режимі вайб-кодингу (AI-асистовано), стек має бути **модерновий, добре задокументований і з великою AI-тренувальною базою** — щоб асистент генерував якісний код з мінімумом помилок.

### Frontend

| Технологія | Призначення | Чому саме вона |
|---|---|---|
| **Next.js 15 (App Router)** | SSR/SSG фреймворк | SEO, оптимізація зображень, API routes |
| **TypeScript** | Типобезпека | Критично для адмінки, менше багів |
| **Tailwind CSS v4** | Стилізація | Швидкість, AI добре знає синтаксис |
| **shadcn/ui** | UI-компоненти | Копіюються в проект, повний контроль |
| **Framer Motion** | Анімації | Мікроанімації, переходи |
| **next-intl** | Локалізація (опційно) | Якщо знадобиться EN/RU |

### Backend + База даних + Сховище

**Supabase** — стратегічний вибір для цього кейсу:

- PostgreSQL (повноцінна реляційна БД);
- вбудована автентифікація для адмінів (magic link без паролів);
- Storage для фото (до/після, аватарки майстрів);
- Row Level Security — прості правила доступу;
- Realtime-підписки (якщо захочете live-оновлення);
- безкоштовний тариф покриває трафік такого сайту з запасом;
- AI-асистенти знають Supabase SDK досконало.

### Хостинг

- **Vercel** для Next.js (безкоштовний тариф тримає такий сайт).
- **Supabase Cloud** для бекенду.

### Допоміжні бібліотеки

| Бібліотека | Для чого |
|---|---|
| **Resend** або **Brevo** | Транзакційні email |
| **Uploadthing** / Supabase Storage | Завантаження зображень |
| **TanStack Query** | Кешування даних на фронті |
| **Zod** | Валідація форм і API (один раз для клієнта і сервера) |
| **React Hook Form** | Робота з формами в адмінці |
| **Tiptap** або **Plate** | Rich text редактор для блогу |
| **date-fns** | Робота з датами (локалізовано українською) |
| **sonner** | Toast-повідомлення |

## 8. Кастомна адмін-панель

Це серце проекту. Розпишу детально.

### 8.1. Загальні принципи

**Мобільна версія обов'язкова.** Адміністратор має змінити ціну або додати фото з телефону між клієнтами. Tailwind + shadcn це дають з коробки.

**Українською мовою.** Всі лейбли, кнопки, повідомлення. Жодної англомовної термінології.

**Превʼю перед публікацією.** Кнопка «Подивитися на сайті» відкриває в новому табі цю саму сторінку з draft-режимом (через Next.js Draft Mode).

**Автозбереження** у драфт (localStorage + кожні 30 сек у БД). Жодних втрат при випадковому закритті вкладки.

**Оптимістичні оновлення.** Клік — миттєво побачила зміну, потім фоном пішло на сервер.

**Візуальна зворотна відповідь** — усі дії підтверджуються toast-повідомленнями («Послугу збережено», «Майстра додано»).

### 8.3. Дашборд

Зверху — 4 картки зі швидкою статистикою:
- Нові заявки (за тиждень)
- Найпопулярніша послуга за записами
- Останнє оновлення Instagram
- Кількість активних послуг / майстрів

Під ними — стрічка «Швидкі дії»:
- «Додати послугу»
- «Додати фото до/після»
- «Створити акцію»
- «Додати відгук»

І нижче — **список нещодавніх заявок** з кнопками «Зателефонувати», «Написати в Telegram», «Позначити як оброблено».

### 8.4. Сторінка «Послуги»

Вгорі фільтри: категорія, активна/прихована, пошук за назвою.

Основна частина — **таблиця з drag-and-drop** для зміни порядку відображення на сайті. Колонки:

| Порядок | Фото | Назва | Категорія | Тривалість | Ціна від | Активна | Дії |
|---|---|---|---|---|---|---|---|
| ≡ | 🖼 | Йорк стандарт комплекс | Комплекси | 2:30 | 650 грн | ✓ | ✏️ 🗑 |

Клік по рядку — повна форма редагування в **слайдовій панелі збоку** (sheet), а не перехід на іншу сторінку.

**Форма редагування послуги:**
- Назва (текст)
- Опис (markdown editor)
- Що входить (теги-пілюльки)
- Категорія (селект)
- Тривалість (число, хвилини)
- Ціна від / до (числа)
- Фото (drag-and-drop завантажувач з preview і кропом)
- Прямий лінк на Easyweek
- Для яких порід (мультиселект)
- Активна / Прихована (toggle)
- Featured (показувати на головній)

Кнопки: «Зберегти», «Зберегти і подивитися», «Дублювати», «Видалити».

### 8.5. Сторінка «Майстри»

Картковий вигляд замість таблиці — для майстрів це природніше. Кожна картка: фото, ім'я, роль-бейдж, кількість послуг, до яких прив'язана, кнопки дій.

Форма редагування майстра містить ще й прев'ю картки, як вона виглядатиме на сайті.

### 8.6. Сторінка «Галерея до/після»

Особливий інтерфейс — **дві кнопки завантаження поруч (Before + After)**. Після завантаження — одразу показується суміжний превʼю.

**Масове завантаження**: закидаєш 20 фото, розподіляєш їх парами через drag-and-drop.

### 8.7. Сторінка «Відгуки»

Головна фішка — можливість **додати відгук зі скріншоту**: завантажуєш скрін з Instagram Stories, адмінка сама його показує на сайті як картку.

Фільтр «Featured» — показувати на головній. Дуже важливий, бо ротацію відгуків треба робити часто.

### 8.8. Сторінка «Блог»

Редактор статей на базі **Tiptap** або **Plate**:
- Форматування тексту
- Вставка зображень з медіатеки
- Вставка YouTube/Instagram embed
- Вставка кнопок «Записатися»

Бокова панель із SEO-полями: title, description, OG image. Внизу — кнопки «Зберегти як чернетку», «Опублікувати», «Запланувати на дату».

### 8.9. Сторінка «Налаштування»

Один довгий скрол з розділами:
- **Контакти** — телефон, адреса, месенджери
- **Робочі години** — таблиця днів тижня
- **Hero-екран** — заголовок, підзаголовок, фото
- **Тексти за замовчуванням**
- **SEO** — дефолтні meta-теги, OG-зображення
- **Інтеграції** — ключі API Instagram, лінк Easyweek, Telegram-бот

### 8.10. Сторінка «Медіатека»

Усі завантажені зображення в одному місці з:
- фільтром (по типу — послуги/майстри/галерея/блог),
- пошуком за назвою,
- можливістю видалити й замінити.

Критично важлива, бо інакше в Supabase Storage накопичується смітник.

### 8.11. Сторінка «Заявки»

Таблиця з усіма заявками з форми «Записатися вперше». Фільтр за статусом. Клік — деталі + кнопки:
- «Зателефонувати»
- «Написати в Telegram/Viber»
- «Створити запис в Easyweek»

## 9. Інтеграції з зовнішніми сервісами

### 9.1. Easyweek

Найімовірніше, у Easyweek немає публічного API для читання записів. Інтеграція через:

**1. Iframe-вбудовування** віджета на сайт. Опція є в налаштуваннях Easyweek.

**2. Deep links** на конкретні послуги/майстрів — зберігаються в полях:
- `services.easyweek_direct_link`
- `masters.easyweek_master_link`

В адмінці це просто текстові поля, куди копіюється URL з панелі Easyweek.

**3. Універсальне посилання** — з `site_settings.easyweek_main_url`.

**4. Webhooks** (якщо тариф дозволяє) — приймати на `/api/webhooks/easyweek` і зберігати в таблицю `bookings` для статистики.

### 9.2. Instagram

**Варіант А: Instagram Graph API** (офіційний, кращий)
- Потрібен Facebook Business акаунт + Instagram Business акаунт
- Довготривалий токен доступу
- Endpoint `/me/media` віддає останні пости
- Токен в Supabase Edge Function як секрет
- Next.js route `/api/instagram/feed` кешує результат на 1 годину
- Токен треба оновлювати раз на 60 днів — cron job

**Варіант Б: Вбудований віджет** (простіше)
- Behold.so або EmbedSocial
- Без контролю над дизайном

Для кастомного сайту варіант А кращий.

### 9.3. Месенджери на сайті

На сайті — плаваюча кнопка справа знизу з розгортанням на:
- Telegram
- Instagram Direct
- Viber
- Телефон

Компонент `<MessengerFab />` у layout — присутній на всіх сторінках.

### 9.4. Telegram-бот для адміна

Supabase Edge Function. Бот надсилає в Telegram-чат адміна:
- Нову заявку з форми «Записатися вперше»
- Новий відгук, доданий клієнтом через форму
- Щоденний дайджест о 9:00 («сьогодні 7 записів»)

Реалізація — 100–150 рядків коду, AI-асистент напише за одну сесію.

Рекомендація — **Umnico** або self-hosted **Chatwoot**.

Це саме по собі економить кілька годин на тиждень.

## 13. Безпека

- **Supabase Auth** з магік-лінком на email (без паролів — надійніше).
- **Row Level Security**:
  - звичайний веб-клієнт читає тільки `active=true`;
  - адмін-клієнт з роллю `service_role` читає все.
- **CORS** для адмін-API обмежений доменом.
- **Rate limiting** на форми (через Upstash Redis).
- **Аудит-лог**: кожна зміна записується в `audit_log` (хто, що, коли змінив).
- **Бекапи** Supabase — щоденні (вбудовано в платний тариф).
- Всі секрети (API ключі) — в environment variables, ніколи в коді.

## 15. Примітки для вайбкодингу

### Файлова структура

```
/app
  /(public)              — публічні сторінки
    /page.tsx            — головна
    /poslugy/page.tsx
    /komanda/page.tsx
    ...
  /admin                 — адмінка
    /layout.tsx          — перевірка авторизації
    /page.tsx            — дашборд
    /services/page.tsx
    ...
  /api
    /webhooks/easyweek
    /instagram/feed
    /telegram/notify
/components
  /ui                    — shadcn компоненти
  /public                — блоки публічного сайту
  /admin                 — компоненти адмінки
  /shared                — спільні (footer, header)
/lib
  /supabase
    /client.ts
    /server.ts
    /admin.ts
  /db                    — функції для роботи з кожною таблицею
    /services.ts         — getServices, createService, ...
    /masters.ts
    ...
  /validations           — Zod схеми
  /utils
/hooks
/types                   — TypeScript types
/public                  — статичні файли
/supabase
  /migrations            — SQL міграції
```

### Правила вайбкодингу

1. **Кожна таблиця = окремий файл** `/lib/db/services.ts` з функціями `getServices()`, `getService(id)`, `createService()`, `updateService()`, `deleteService()`.
2. **Admin routes захищені через middleware**, що перевіряє сесію Supabase.
3. **Public routes** по можливості Static з ISR (revalidate раз на годину) — для швидкості та SEO.
4. **Валідація одна й та сама на клієнті й сервері** через Zod.
5. **Міграції БД** — зберігати в `/supabase/migrations` як SQL-файли.
6. **Компоненти серверні за замовчуванням** (Next.js 15 pattern), клієнтські тільки там де потрібна інтерактивність.
7. **Форми** завжди через React Hook Form + Zod resolver.
8. **Зображення** через `next/image` для оптимізації.
9. **Всі строки для UI** — в одному файлі `/lib/strings.ts` для легкої зміни.
10. **Коміти по фічах**, не по файлах. Гілка `main` завжди deployable.

### Стратегія роботи з AI-асистентом

- Починати з **створення таблиць Supabase** (один промпт = одна таблиця з міграцією).
- Далі **серверні функції читання** (одна функція = один промпт).
- Після цього **компоненти публічного сайту** (один блок = один промпт).
- Адмінка робиться **після** того, як публічний сайт показує моковані дані.
- **Тести** — мінімальні, тільки критичні функції (авторизація, оплата).

## Підсумок

Цей сайт має закрити три задачі:

1. **Замінити Instagram-only присутність** повноцінним веб-представництвом з SEO.
2. **Мінімізувати ручне адміністрування** через кастомну адмінку і автоматичні інтеграції.
3. **Збільшити конверсію** з відвідувачів у записи за рахунок прямих лінків Easyweek і прозорого прайсу.

При запуску V1 салон отримує:
- сайт, що оновлюється самостійно без залазить у код;
- Instagram-стрічку, яка синхронізується сама;
- прямий запис через Easyweek без посередників;
- єдиний inbox для всіх месенджерів (зовнішній інструмент).

Тижнева рутина після запуску: **15–30 хвилин** на перевірку заявок та дрібні правки контенту.
